很好 👍 现在的报错非常清晰，是**方法签名不匹配问题**：

```
java: cannot find symbol 
  symbol:   method login(com.sky.dto.EmployeeLoginDTO)
  location: variable employeeService of type com.sky.service.EmployeeService
```

意思是：

> 你在 EmployeeController 里调用了
> `employeeService.login(EmployeeLoginDTO dto)`
> 但是 **EmployeeService 接口里没有这个方法**

---

## ✅ 你现在的 Controller 调用是这样的

```java
Employee employee = employeeService.login(employeeLoginDTO);
```

所以，必须保证：

### 👉 EmployeeService 接口里声明 login 方法

### 👉 EmployeeServiceImpl 中实现 login 方法

---

## ✅ 第一步：修改 EmployeeService 接口（重点）

位置：

```
sky-server / src / main / java / com / sky / service / EmployeeService.java
```

补充方法：

```java
import com.sky.dto.EmployeeLoginDTO;
import com.sky.entity.Employee;

public interface EmployeeService {

    /**
     * 员工登录
     */
    Employee login(EmployeeLoginDTO employeeLoginDTO);

    // 你原来可能还有其他方法，比如 save / update 等
}
```

---

## ✅ 第二步：在实现类中真正实现 login 方法

位置：

```
EmployeeServiceImpl.java
```

例子（典型苍穹外卖写法）

```java
@Service
public class EmployeeServiceImpl implements EmployeeService {

    @Autowired
    private EmployeeMapper employeeMapper;

    @Override
    public Employee login(EmployeeLoginDTO dto) {

        //1 查询数据库
        Employee employee = employeeMapper.getByUsername(dto.getUsername());

        //2 账号不存在
        if (employee == null) {
            throw new LoginFailedException("账号不存在");
        }

        //3 密码错误
        if (!employee.getPassword().equals(dto.getPassword())) {
            throw new LoginFailedException("密码错误");
        }

        //4 账号被锁定
        if (employee.getStatus() == 0) {
            throw new LoginFailedException("账号被锁定");
        }

        return employee;
    }
}
```

> 上面逻辑你可先随便 return，先保证方法存在即可
> 业务细节以后慢慢补

---

## ❌ 为什么你会报错？

典型三种情况之一：

1. 你只写了 **EmployeeServiceImpl.login()**
   但 👇
   ➜ 忘了在 interface 中声明

2. 你方法名写成了 loginXXX / doLogin 等

3. DTO 类型不一致，比如：

```
login(EmployeeDTO dto)   ❌
login(EmployeeLoginDTO dto) ✔
```

## 🎯 顺便送你一句**面试关键词总结**

> ❝ Controller 依赖的是接口，不是实现类
> 所以接口方法签名必须与调用完全一致。
> 如果实现类有方法但接口没有声明，Spring 仍然认为“方法不存在”。 ❞

这就是你现在遇到的问题 👆
