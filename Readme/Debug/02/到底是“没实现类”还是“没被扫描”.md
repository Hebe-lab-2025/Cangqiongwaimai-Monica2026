你现在**编译已经过了**，Spring Boot 启动失败是因为：

> `CategoryController` 里 `@Autowired CategoryService`，但 Spring 容器里**没有任何** `CategoryService` 的 Bean。
> 所以报：`No qualifying bean of type 'com.sky.service.CategoryService'`

---

## 1) 先用 10 秒定位：到底是“没实现类”还是“没被扫描”

去项目里搜这两个东西：

### A. 接口存在吗？

路径通常是：

* `sky-server/src/main/java/com/sky/service/CategoryService.java`

### B. 实现类存在吗？

你应该能搜到类似：

* `CategoryServiceImpl`（一般在 `com.sky.service.impl`）

实现类上必须有 **`@Service`**（或 `@Component`）：

```java
@Service
public class CategoryServiceImpl implements CategoryService { ... }
```

---

## 2) 最常见的 3 个原因 + 对应修法

### ✅ 原因 1：实现类没加 `@Service`

**现象**：有 `CategoryServiceImpl`，但没注解
**修法**：给实现类加 `@Service`

---

### ✅ 原因 2：实现类包名不在 Spring 扫描范围

你启动类是 `com.sky.SkyApplication`。

Spring Boot 默认扫描：**启动类所在包 `com.sky` 及其子包**。

如果你的实现类跑到了比如：

* `cn.itcast...`
* `com.itheima...`
* 或者不在 `com.sky.*`

那就扫描不到。

**修法 A（推荐）**：把代码包名挪回 `com.sky.service.impl`
**修法 B**：在启动类加扫描范围：

```java
@SpringBootApplication(scanBasePackages = "com.sky")
public class SkyApplication { ... }
```

> 注意：如果你有多个根包，就把它们都加进去。

---

### ✅ 原因 3：你用的是 MyBatis Mapper 注入，但 Mapper 没被扫描导致 Service 创建失败

有时真正问题不是 Service 没写，而是 **ServiceImpl 里注入了 Mapper**，Mapper 没被扫描，导致 ServiceImpl Bean 根本创建不出来；最后 controller 才报 “找不到 CategoryService”。

**你可以在日志里往上翻**，看是否还有类似：

* `No qualifying bean of type 'CategoryMapper'`
* 或 `Error creating bean with name 'categoryServiceImpl'`

**修法**：确保 MyBatis 扫描了 mapper 包：

在启动类加：

```java
@MapperScan("com.sky.mapper")
@SpringBootApplication
public class SkyApplication { ... }
```

并确保 `CategoryMapper` 在 `com.sky.mapper` 下，且有：

```java
@Mapper
public interface CategoryMapper { ... }
```

（通常 `@MapperScan` 有了就不需要每个 mapper 写 `@Mapper`）

---

## 3) 你现在立刻能做的检查（不需要猜）

在 IDEA 全局搜：

* `interface CategoryService`
* `class CategoryServiceImpl`
* `@Service`
* `package com.sky.service.impl`
* `@MapperScan`
* `interface CategoryMapper`

如果你搜不到 `CategoryServiceImpl`，那就说明你项目里**压根缺实现类**（需要补）。

---

## 4) 最小可运行兜底（临时验证用）

如果你只是想先让项目跑起来（不做业务），可以先写一个空实现：

```java
@Service
public class CategoryServiceImpl implements CategoryService {
    // 先返回空/默认值，保证能启动
}
```

但更推荐按上面把真实实现类/扫描问题修好。
