你这个 **`%`**，在你现在的上下文里，**只有一个核心含义**，而且非常重要。

---

## 一句话结论

> **`%` 是 SQL 里的通配符，表示“任意长度的任意字符”。**

---

## 在 SQL / MyBatis 里 `%` 是干嘛的？

### 1️⃣ 用在 `LIKE` 里（最常见）

```sql
name LIKE '%Tom%'
```

含义是：

> 名字里 **只要包含 `Tom`** 就匹配

---

## `%` 的三种典型用法（一定要记）

| 写法        | 含义       |
| --------- | -------- |
| `'%Tom%'` | 包含 Tom   |
| `'Tom%'`  | 以 Tom 开头 |
| `'%Tom'`  | 以 Tom 结尾 |

---

## 结合你刚才的 MyBatis SQL

你之前的代码是：

```xml
and name like concat('%', #{name}, '%')
```

等价于最终 SQL：

```sql
and name like '%张%'
```

👉 **模糊查询**

---

## 为什么不能直接写成这样？（新手常犯）

### ❌ 错误写法

```xml
and name like '%#{name}%'
```

这是错的，因为：

* `#{name}` 是 **占位符**
* MyBatis 不会替换字符串内部

---

### ✅ 正确写法（你用的）

```xml
and name like concat('%', #{name}, '%')
```

或者：

```xml
and name like CONCAT('%', #{name}, '%')
```

---

## `%` 和 `_` 的区别（补充）

| 符号  | 含义            |
| --- | ------------- |
| `%` | 任意长度字符（0 个也行） |
| `_` | **一个** 任意字符   |

例子：

```sql
name LIKE '_om'
```

👉 匹配：Tom、Mom
👉 不匹配：From

---

## 面试一句话版（直接背）

> 在 SQL 中，`%` 是 LIKE 查询的通配符，表示任意长度的字符，常用于模糊查询，例如 `%keyword%` 表示包含该关键字。
