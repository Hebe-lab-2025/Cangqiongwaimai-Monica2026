# 📌 Markdown 中的「绝对路径 vs 相对路径」

在 `README.md` 或文档里引用图片、文件时，**路径写法非常重要**，主要分为两种：

---

## 一、绝对路径（Absolute Path）

### ✅ 定义

> **从“最顶层位置”开始写的完整路径**
> 不依赖当前文件所在位置

在 GitHub 中，绝对路径通常是 **完整的 URL**

---

### ✅ 示例（你给的链接）

```md
![Spring Controller 接参](https://github.com/Hebe-lab-2025/Cangqiongwaimai-Monica2026/blob/main/Readme/Springboot/images/Spring%20Controller%20%E6%8E%A5%E5%8F%82.png)
```

### 📌 特点

* ✅ 在 **任何地方都能显示**
* ✅ GitHub、Notion、博客、Issue 都能用
* ❌ URL 很长
* ❌ 仓库结构变了，链接容易失效

---

### 🧠 适合场景

* README 首页
* 对外文档 / 博客
* Issue / PR 描述
* 不确定文件引用位置时

---

## 二、相对路径（Relative Path）

### ✅ 定义

> **相对于当前 `.md` 文件的位置来写路径**

📍 不写仓库名、不写 `main`、不写 `https://github.com`

---

### ✅ 假设你的目录结构是：

```text
Cangqiongwaimai-Monica2026
├── Readme
│   └── Springboot
│       ├── README.md   👈 当前 md 文件
│       └── images
│           └── Spring Controller 接参.png
```

---

### ✅ 相对路径写法

```md
![Spring Controller 接参](images/Spring%20Controller%20%E6%8E%A5%E5%8F%82.png)
```

### 📌 特点

* ✅ 路径短、干净
* ✅ 仓库内移动不容易坏
* ❌ 只能在 **当前仓库结构下** 正常显示
* ❌ 拷到别的地方可能失效

---

### 🧠 适合场景

* 项目内 README
* 技术笔记仓库
* 长期维护的文档
* GitHub Pages

---

## 三、`../` 是什么意思？（非常重要）

```md
![xxx](../images/demo.png)
```

### 含义

> `../` = 回到**上一级目录**

📌 常见用法：

| 写法                | 含义            |
| ----------------- | ------------- |
| `images/a.png`    | 当前目录下的 images |
| `../images/a.png` | 上一级目录的 images |
| `../../img/a.png` | 上上级目录         |

---

## 四、绝对 vs 相对 一眼对比表

| 对比项        | 绝对路径 | 相对路径 |
| ---------- | ---- | ---- |
| 是否依赖当前文件位置 | ❌ 否  | ✅ 是  |
| URL 长度     | ❌ 长  | ✅ 短  |
| 跨平台/跨仓库    | ✅ 强  | ❌ 弱  |
| 仓库内维护      | ❌ 差  | ✅ 好  |
| 推荐使用场景     | 对外   | 项目内  |

---

