## 1️⃣ 一句话结论（先记）

> **LangChain 是一个“把大模型当组件来编程”的开发框架**，用来快速搭建 **LLM 应用**（聊天、RAG、Agent、工具调用）。

---

## 2️⃣ 它解决什么问题？

直接用大模型 API 的痛点：

* Prompt 写一堆，难复用
* 多步推理要自己 glue code
* 检索、记忆、工具调用很乱

👉 **LangChain 把这些“套路”封装成模块**。

---

## 3️⃣ 核心组成（面试必讲 4 件）

### ① LLM / ChatModel

* 对接 OpenAI、Anthropic、Bedrock 等
* 统一接口

### ② Prompt Template

```python
PromptTemplate("用{language}解释{topic}")
```

* 模板化 prompt，方便复用

### ③ Chain（重点）

```
Prompt → LLM → Parser → 下一步
```

* 把多步逻辑串起来

### ④ Memory

* 维护对话上下文（Buffer / Summary）

---

## 4️⃣ 最火的用法：RAG（检索增强生成）

```
用户问题
   ↓
向量检索（DB / 文档）
   ↓
相关内容
   ↓
LLM 生成答案
```

LangChain 提供：

* 文档加载器（PDF / Web / DB）
* Text Splitter
* Embeddings
* Vector Store（FAISS / Pinecone）

---

## 5️⃣ Agent（进阶）

> **模型能“自己决定用哪个工具”**

例如：

* 搜索
* 查数据库
* 调 API

LangChain 提供：

* Tool
* Agent Executor
* ReAct / Function Calling

---

## 6️⃣ LangChain 适合什么人？

✅ 想快速做 LLM Demo
✅ 做 RAG / Chatbot
❌ 追求极致性能 / 完全可控（有时太重）

---

## 7️⃣ 优缺点（面试常问）

### 👍 优点

* 快速上手
* 生态全
* RAG/Agent 开箱即用

### 👎 缺点

* 抽象多、调试复杂
* 版本变动快
* 有时性能不如手写

---

## 8️⃣ LangChain vs 直接写 API

| 对比  | LangChain | 直接 API |
| --- | --------- | ------ |
| 上手  | 快         | 慢      |
| 灵活性 | 中         | 高      |
| 维护  | 中         | 高      |
| 适合  | Demo / 产品 | 底层系统   |

---

## 9️⃣ 面试一句话标准答

> LangChain 是一个 LLM 应用开发框架，通过 Chain、Memory、Tool 等抽象，帮助开发者快速构建聊天、RAG 和 Agent 系统。
